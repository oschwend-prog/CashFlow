### 4. Mapbox editorial style

A new `applyEditorialMapStyle(m)` function runs post-load on every Mapbox instance. It iterates layers and applies:

- **Land**: `#050A12` (near-ink)
- **Water**: `#0A111E` at 92% opacity
- **Land use (parks/sand/wood)**: `#0D1623` at 40% opacity
- **Buildings**: `#0D1623` at 55% opacity with paper-alpha 6% outline
- **Roads**: `rgba(242,237,215,0.16)` strokes, widths reduced to 35%
- **Bridges/tunnels/rails**: paper-alpha 8%
- **POI / transit / airport layers**: hidden entirely
- **Text labels**: paper-dim at 75% opacity with ink halo; minor/suburb/street labels further dimmed to 45%

The base style is still `mapbox://styles/mapbox/dark-v11` тАФ the function patches paint properties in-place after the style loads. No external style URL needed, works with the existing Mapbox token.

## What's NOT in v8.6 yet (honest remaining list)

1. **Chat is not backend-connected.** Wire to Supabase realtime channel keyed on `order.id`. Present as a prompt engineering task: "Wire `sendChat()` to Supabase realtime, subscribe on render, unsubscribe on unmount/go."
2. **Custom-published Mapbox style.** Current approach patches paint properties in-memory. For a fully custom style URL hosted on Mapbox Studio, you'd need to bake the paint properties into a style JSON and publish it. The in-memory approach works and looks right, it just runs on every map load.
3. **Activity / Wallet / Notifications / Disputes full-screen sheets.** The `.fs-*` scaffolding is v8.6 but internal content blocks may still have one-off inline styles that slipped through the pattern match.
4. **Instant badge `тЪб`.** Preserved but on iOS may render as orange emoji rather than text glyph. True fix: add `font-variant-emoji: text` on the `.tag-instant` class. Low priority.
5. **Heatmap / sparkline SVG colors.** Earnings screen heatmap cells and the 7-day sparkline use `var(--em)` тАФ already correct. No action.
6. **renderSubRole, renderCashPoint, renderRequest, renderCourierSelect.** Legacy intermediate screens. Inherit the v8.6 global override layer so they look acceptable, but haven't received targeted markup rewrites. Flag if anything reads wrong.
