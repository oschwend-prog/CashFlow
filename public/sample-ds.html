Mode Theming § 02

One variable. Ten outcomes.

The --accent CSS variable switches based on body.supply-mode. Default is paper cream (Get Cash). Toggling triggers emerald re-themeing across ~10 elements in 300ms: § glyphs, preset states, address icon, CTA background, HOME tab, sheet glow, user pin.

// Default — Get Cash mode
:root {
--accent: var(--paper);
}

// Supply Cash mode
body.supply-mode {
--accent: var(--em);
}

The Ten Rules § 06

Rules of restraint.

Principles that make the system cohesive. Each rule has been earned through iteration.

  1. One italic emerald accent word per screen. No more. Signature, not decoration.
  2. Emerald #00E8B8 only for active state or meaningful accent. Never decorative.
  3. Every numerical value is a tombstone: Instrument Serif with optional emerald hairline underline.
  4. Labels are IBM Plex Mono · uppercase · letter-spacing .22em. Always.
  5. Dividers are hairlines (1px rgba(242,237,215,.12)). Never borders, never boxes.
  6. Currency is rendered as £100 with the £ as a small serif superscript, never same-size.
  7. Corner L-brackets reserved for certificate moments only (receipt, hero stats, hero cards).
  8. CTAs are paper-cream rectangles with emerald slide-fill hover. One primary per screen.
  9. No emoji in production. Use Unicode ornaments (◉ ◈ § ⌂ ⚿) or custom SVG.
  10. Film grain at 4–5% opacity with mix-blend-mode: overlay. Subtle, not textural.
